# @frouvel

- frourio + Laravelの造語です.
- TypeScriptフルスタックフレームワークでは, 依然としてベストプラクティスが確立されていない状態です. Laravelのような開発者の共通認識として成り立つような設計思想として具現化することを目指します.
  - 設計で困ったら, Laravelならどうするか？に回帰することを開発者に強制するとも言えます.
  - 特定の開発者に固有の組み方ではなく, 「フレームワークならどうするか？」を常に考えてもらうためのレイヤとも言えるでしょう.
  - また, AI生成のやりやすさに一役買うはずです. フルスタックTypeScriptのコード事例は少ないですし, よくできたフレームワークもそんなにないですが, Laravelのエコシステムとコードサンプルは驚くほど存在します.
- 同様に@frouri-on-railsもできるのかもしれません(笑).
